# 一、课程动机：为什么需要MLP语言模型？

## 1.1 回顾上节课：二元语法模型（Bigram）

上节课实现的二元语法模型逻辑简单，核心是：

- 输入：前1个字符

- 输出：下一个字符的概率分布

- 训练：统计"前字符→后字符"的出现次数，归一化得到27×27的概率矩阵（26个字母+1个结束符"."）

## 1.2 二元语法模型的致命缺陷（核心痛点）

这也是我们必须转向MLP的根本原因，两个核心问题：

1. 上下文太短，预测效果极差：仅能看到前1个字符，忽略更长的上下文信息。例如，仅看"e"无法判断下一个字符，但结合"emm"上下文，下一个字符大概率是"a"（如Emma）。

2. 参数指数爆炸（维度灾难）：若强行增加上下文长度（如三元、四元语法），参数数量会呈指数增长：

    - 二元语法：27×27 = 729个参数

    - 三元语法：27³ = 19683个参数

    - 八元语法：27⁸ ≈ 2800亿个参数，完全无法训练和存储

## 1.3 本节课核心目标

解决上述矛盾：既要增加上下文长度（如看前3个字符预测下一个），又要避免参数指数爆炸，解决方案是复刻Bengio等人2003年提出的MLP语言模型。

# 二、核心思想：Bengio et al. 2003 MLP语言模型

本节课完全复刻经典论文《A Neural Probabilistic Language Model》，核心是3个关键思想，也是MLP语言模型的核心逻辑，同时解答我曾提出的"嵌入层、MLP结构"相关疑问。

## 2.1 思想1：固定长度上下文（block_size）

不再关注无限前序字符，而是固定看前k个字符（本节课k=3，即block_size=3）预测下一个字符，示例：

名字"emma"→ 补充开始/结束符后为".emma."，拆分为以下样本：

- [., ., .] → 预测 e

- [., ., e] → 预测 m

- [., e, m] → 预测 m

- [e, m, m] → 预测 a

- [m, m, a] → 预测 .

优势：上下文长度可控，既能利用局部序列信息，又避免参数爆炸。

## 2.2 思想2：字符嵌入（Embedding）—— 彻底解析（重点）

这是解决"离散字符无法被神经网络处理"的核心，也是我曾重点提问的内容，详细拆解如下：

### 2.2.1 传统表示（one-hot编码）的缺陷

27个字符（a-z + .）用one-hot编码时，每个字符是27维稀疏向量（如a→[1,0,...,0]，.→[0,...,1]），存在3个问题：

- 过于稀疏，大部分元素为0，浪费内存；

- 字符间无关联，如元音a和e的向量完全正交，模型无法识别它们的相似性；

- 维度过高，若扩展到单词级别，维度会飙升至几万维。

### 2.2.2 嵌入层的定义与作用

核心：给每个字符分配一个低维、连续的向量，该向量由模型训练学习得到，而非人工定义，本质是一个"查找表"。

本节课具体设置（与代码对应）：

- 字符总数（vocab_size）：27

- 嵌入维度（C）：初始为2，后续调优至10（提升模型表达能力）

- 嵌入矩阵（W_emb）：形状为(27, C)，每一行对应一个字符的向量

示例（C=2时）：a→[0.3, -0.1]，e→[0.28, -0.11]（元音向量靠近），x→[-0.5, 0.9]（与元音远离）。

### 2.2.3 嵌入层的使用方式

输入为字符索引（0~26），通过查表直接获取对应向量，PyTorch代码实现：

```python
import torch
embedding = torch.nn.Embedding(vocab_size, C)  # 27个字符，每个映射为C维向量
x_emb = embedding(torch.tensor([5]))  # 输入字符索引5，直接查表获取向量
```

关键：嵌入层本质是"one-hot向量 × 嵌入矩阵"的等价操作，但查表更高效、省内存，无需进行冗余的矩阵乘法。

### 2.2.4 嵌入层的核心意义

- 将离散符号（字符）转换为连续向量，让神经网络可处理；

- 相似字符（如元音、辅音）在向量空间中会自动聚类；

- 维度可控（C=2、10、64均可），避免维度灾难。

## 2.3 思想3：MLP（多层感知机）统一建模（重点）

结合固定上下文和字符嵌入，用MLP完成"上下文→下一个字符"的预测，解决参数爆炸问题，同时解答我曾提问的"H（隐藏层神经元数）的含义"。

### 2.3.1 核心流程（对应代码前向传播）

1. 输入：3个字符的索引（x1, x2, x3，每个0~26）；

2. 嵌入：每个字符查嵌入表，得到3个C维向量（v1, v2, v3）；

3. 拼接/展平：将3个C维向量合并为1个3C维长向量（x_concat）；

4. 隐藏层：通过线性变换+非线性激活，提取特征；

5. 输出层：映射到27个字符的未归一化概率（logits）；

6. 损失计算：用交叉熵损失，让正确字符的概率最大。

### 2.3.2 关键参数解析（重点解答我的疑问）

H（隐藏层神经元数）：是我们手动设定的超参数，不是模型训练出来的，本节课初始默认H=200，后续可调优（如100、300）。

- H的作用：决定隐藏层的特征提取能力，H越大，模型容量越强；

- H的影响：

    - H太小（如10）：模型容量不足，欠拟合，学不到复杂字符规律；

    - H太大（如1000）：模型容量过剩，容易过拟合，泛化能力变差；

    - H=200：针对本节课"名字数据集"的折中最优值，兼顾拟合能力和泛化能力。

### 2.3.3 各层维度对应（必记）

- 嵌入后：(N, 3, C) → N个样本，每个样本3个字符，每个字符C维向量；

- 展平后：(N, 3C) → 每个样本合并为3C维长向量；

- 隐藏层权重（W1）：(3C, H) → 输入3C维，输出H维；

- 隐藏层偏置（b1）：(H,) → 匹配隐藏层神经元数；

- 输出层权重（W2）：(H, 27) → 输入H维，输出27维（对应27个字符）；

- 输出层偏置（b2）：(27,) → 匹配27个字符。

### 2.3.4 MLP的核心优势（解决参数爆炸）

对比传统n-gram模型，MLP的参数数量仅线性增长，而非指数增长：

- 三元语法：参数约27³=19683；

- MLP：参数=嵌入层（27×C）+ 隐藏层（3C×H）+ 输出层（H×27），总参数仅几万，即使block_size增加，参数也只是线性增加。

# 三、数据集构建（代码落地）

核心目标：将原始名字文本，转换为MLP可处理的"上下文→目标字符"训练样本，步骤如下：

## 3.1 字符与索引的映射

先将字符转换为模型可处理的数字，定义两个映射字典：

```python
words = open('names.txt', 'r').read().splitlines()
chars = sorted(list(set(''.join(words))))
stoi = {s:i+1 for i,s in enumerate(chars)}  # 字符→索引，a-z对应1~26
stoi['.'] = 0  # 结束符单独分配索引0
itos = {i:s for s,i in stoi.items()}  # 索引→字符，反向映射，用于后续生成名字
```

## 3.2 滑动窗口生成样本

基于block_size=3，用滑动窗口遍历每个名字，生成"上下文-目标"样本，以名字"emma"为例：

1. 补充开始/结束符：emma → .emma.；

2. 转换为索引：.emma. → [0, 5, 13, 13, 1, 0]（e=5，m=13，a=1，.=0）；

3. 滑动窗口采样（每次取3个字符为上下文，下一个为目标）：

|上下文（3个字符）|目标字符|X样本（上下文索引）|Y样本（目标索引）|
|---|---|---|---|
|[., ., .]|e|[0, 0, 0]|5|
|[., ., e]|m|[0, 0, 5]|13|
|[., e, m]|m|[0, 5, 13]|13|
|[e, m, m]|a|[5, 13, 13]|1|
|[m, m, a]|.|[13, 13, 1]|0|

## 3.3 最终数据集格式

处理所有名字后，得到两个PyTorch张量：

- X：形状为(N, 3)，N为总样本数（本节课约228146），每个样本是3个字符的索引；

- Y：形状为(N,)，每个元素是对应上下文的下一个字符索引。

```python
X = torch.tensor(X)  # shape: (228146, 3)
Y = torch.tensor(Y)  # shape: (228146,)
```

## 3.4 关键说明

用"."填充开头和结尾，是为了让模型学到名字的开头/结尾模式（如名字开头多为元音或常见辅音，结尾常为元音或n），同时将变长的名字统一为固定长度的输入，方便批量处理。

# 四、模型代码实现（完整落地）

按"嵌入层→展平→隐藏层→输出层→损失计算"的顺序，完整实现模型，结合我曾提问的"展平操作、维度变化"等知识点。

## 4.1 导入库与定义超参数

```python
import torch
import torch.nn.functional as F

# 超参数（均为手动设定，H的含义已明确）
block_size = 3    # 上下文长度
C = 2             # 初始嵌入维度，后续调优至10
vocab_size = 27   # 字符总数
H = 200           # 隐藏层神经元数（自定义超参数）
```

## 4.2 嵌入层实现与维度变化（重点）

这是我曾疑问的"(N,3)→(N,3,C)"维度变化的核心实现：

```python
# 创建嵌入层（查找表）
embedding = torch.nn.Embedding(vocab_size, C)

# 嵌入层维度变换：(N, 3) → (N, 3, C)
X_emb = embedding(X)  # X.shape=(228146,3) → X_emb.shape=(228146,3,2)
```

解释：每个样本从"3个数字（字符索引）"转换为"3个C维向量"，例如输入样本[0,0,5]（..e），嵌入后为[[0.1,-0.2], [0.1,-0.2], [0.3,0.5]]。

## 4.3 嵌入向量展平（衔接MLP输入）

MLP的全连接层只能接收一维向量，因此需要将(N,3,C)展平为(N, 3*C)，这里使用view()操作（课程重点强调）：

```python
# 展平操作：将最后两个维度(3,C)合并为3*C
X_concat = X_emb.view(-1, block_size * C)  # shape: (228146, 6)（C=2时）
```

关键疑问解答：为什么用view()而不是拼接？

- view()零成本：不复制数据，仅改变张量的"查看方式"，效率远高于concat；

- 效果等价：和将3个向量首尾拼接的结果完全一致，适合大批量数据训练。

## 4.4 搭建MLP隐藏层与输出层

```python
# 初始化隐藏层参数（W1, b1）和输出层参数（W2, b2）
W1 = torch.randn((block_size * C, H), requires_grad=True)  # (6, 200)
b1 = torch.randn(H, requires_grad=True)                    # (200,)
W2 = torch.randn((H, vocab_size), requires_grad=True)      # (200, 27)
b2 = torch.randn(vocab_size, requires_grad=True)          # (27,)

# 前向传播（完整流程）
h = torch.tanh(X_concat @ W1 + b1)  # 隐藏层：线性变换 + tanh激活，shape=(N,200)
logits = h @ W2 + b2                # 输出层：线性变换，shape=(N,27)，未归一化概率
loss = F.cross_entropy(logits, Y)   # 交叉熵损失：softmax + 负对数似然，数值稳定
```

关键说明：

- tanh激活函数：引入非线性，让模型能学习复杂的字符规律，没有激活函数则退化为线性模型，无法拟合复杂数据；

- logits：27个数字，对应27个字符的未归一化概率，后续通过softmax转换为归一化概率；

- F.cross_entropy：PyTorch官方实现，一步完成softmax和负对数似然计算，比手动实现更高效、更稳定，避免指数运算溢出。

# 五、模型训练循环（核心流程）

训练的核心是"前向传播→反向传播→参数更新"，共4个固定步骤，同时解答我曾疑问的"为什么要清空梯度、小批量训练的作用"等问题。

## 5.1 打包可学习参数

```python
# 打包所有可学习参数（嵌入层权重 + 隐藏层/输出层参数）
parameters = [embedding.weight, W1, b1, W2, b2]
# 统计总参数数量（课程中用于验证模型规模）
print(f"总参数数量: {sum(p.nelement() for p in parameters)}")
```

## 5.2 定义训练超参数

```python
learning_rate = 0.1  # 学习率（步长），后续会调优
max_steps = 10000    # 训练迭代次数
batch_size = 32      # 小批量大小，深度学习标配
```

## 5.3 核心训练循环（4步）

```python
for i in range(max_steps):
    # 步骤1：随机采样小批量数据（解答：为什么用小批量？）
    ix = torch.randint(0, X.shape[0], (batch_size,))  # 随机选32个样本索引
    X_batch = X[ix]  # 小批量输入：(32, 3)
    Y_batch = Y[ix]  # 小批量标签：(32,)
    # 小批量作用：速度快、省内存，避免全量数据训练的冗余，同时保证梯度稳定
    
    # 步骤2：前向传播，计算小批量损失
    X_emb = embedding(X_batch)               # (32, 3, 2)
    X_concat = X_emb.view(-1, block_size*C)  # (32, 6)
    h = torch.tanh(X_concat @ W1 + b1)       # (32, 200)
    logits = h @ W2 + b2                     # (32, 27)
    loss = F.cross_entropy(logits, Y_batch)  # 计算当前批次损失
    
    # 步骤3：反向传播，计算梯度（解答：为什么要清空梯度？）
    for p in parameters:
        p.grad = None  # 清空上一次的梯度，避免梯度累积导致训练不稳定
    loss.backward()    # 自动计算所有参数的梯度（方向+大小）
    
    # 步骤4：梯度下降，更新参数
    for p in parameters:
        p.data += -learning_rate * p.grad  # 沿损失减小的方向微调参数
    
    # 打印训练进度
    if i % 1000 == 0:
```

## 6.3 拟合状态判断（核心技巧）

通过对比训练损失（train loss）和验证损失（val loss），判断模型拟合状态：

- train loss 高、val loss 高 → 欠拟合：模型容量不足、训练不够或学习率太低；

- train loss 低、val loss 高 → 过拟合：模型背下了训练数据，泛化能力差；

- train loss 低、val loss 低 → 最优状态：模型既学到了规律，又能泛化到新数据。

## 7.1 学习率调优（最关键的超参数）

学习率（lr）是参数更新的步长，直接决定训练效果，课程采用"指数级搜索"的方法找最优学习率：

```python
# 指数级搜索学习率：从0.001到1变化

```

调优结果：最优学习率区间为0.1~0.01，课程最终采用"学习率衰减"策略：

- 前期：用较大学习率（0.1），让损失快速下降，高效收敛；

- 后期：用较小学习率（0.01），精细优化参数，避免训练震荡。

形象比喻：学习率像下楼梯的步子，步子太大易踩空（不收敛），步子太小爬得慢，先大步后小步最优。

# 九、模型采样生成名字（推理过程）

训练好模型后，核心用途是自动生成新名字，流程遵循"循环采样+滑动上下文"，解答我曾关注的"采样逻辑"。

## 9.1 采样核心逻辑

- 初始上下文：[0, 0, 0]（3个结束符"."，代表名字开头）；

- 循环步骤：用当前3个字符预测下一个字符的概率分布 → 按概率随机采样下一个字符 → 更新上下文（丢弃最旧字符，追加新字符）；

- 终止条件：采样到结束符"."（索引0），停止生成。

## 9.2 采样代码实现（课程原版）

```python
# 生成20个新名字
for _ in range(20):
    out = []
    context = [0] * block_size  # 初始上下文：3个"."
    
    while True:
        # 前向传播（推理，无需计算梯度）
        x_emb = embedding(torch.tensor([context]))  # (1, 3, 10)（C=10）
        x_concat = x_emb.view(1, -1)                # (1, 30)
        h = torch.tanh(x_concat @ W1 + b1)          # (1, 200)
        logits = h @ W2 + b2                        # (1, 27)
        
        # 计算归一化概率
        probs = F.softmax(logits, dim=1)           # (1, 27)，概率和为1
        
        # 按概率采样下一个字符（解答：为什么不用argmax？）
        ix = torch.multinomial(probs, num_samples=1).item()
        # multinomial采样：按概率随机选，生成的名字多样、自然；argmax：选概率最大的，名字死板、重复
        
        # 更新上下文（滑动窗口）
        context = context[1:] + [ix]
        
        # 记录生成的字符
        out.append(ix)
        
        # 遇到结束符停止
        if ix == 0:
            break
    
    # 转换为字符，打印生成的名字
    print(''.join(itos[i] for i in out))
```

## 9.3 生成效果对比

- 二元语法模型：生成的名字短、生硬（如em、ma、an）；

- MLP模型（C=10, H=200）：生成的名字更长、更接近真实姓名（如emma、mia、luca），证明模型学到了有效的字符规律。
